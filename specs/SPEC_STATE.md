# SPEC: Durable State

**Layer:** L5 — Durable State
**Owner:** `app/state/`

The Durable State layer makes the agent's plans, decisions, and audit trail survive process restarts. It wraps LangGraph's `SqliteSaver` (for conversational checkpoints) and a thin SQLite repository facade (for refunds, escalations, incidents, and pending approvals). Every write is parameterized, every domain write is idempotent on a stated business key, and every write emits a `LayerEvent`.

## Contract

```python
from app.state import get_checkpointer, get_repository
```

- `get_checkpointer() -> SqliteSaver` — singleton; backed by `settings.sqlite_full_path`. Reuses the same connection pool as `get_repository()`.
- `get_repository() -> Repository` — facade with the following methods:

```python
class Repository:
    def save_refund(self, refund: RefundRecord) -> RefundRecord: ...
    def find_refund(self, conversation_id: str, order_id: str) -> RefundRecord | None: ...
    def save_escalation(self, esc: EscalationRecord) -> EscalationRecord: ...
    def save_incident(self, inc: IncidentRecord) -> IncidentRecord: ...
    def list_pending_approvals(self) -> list[PendingApproval]: ...
    def resolve_approval(
        self,
        approval_id: str,
        resolution: Literal["approved", "denied"],
        approver: str,
    ) -> PendingApproval: ...
    def apply_migrations(self) -> None: ...
```

The Repository is a thin facade — no business logic, no derivation, no merge semantics. It is the only writer to the SQLite database file.

## New domain models

Added to `app/domain/models.py` (with their own model tests in `tests/test_domain_models.py`):

- `RefundRecord` — `refund_id: str`, `conversation_id: str`, `order_id: str`, `customer_id: str`, `amount_usd: float`, `kind: RefundDecisionKind`, `cited_clauses: list[str]`, `reasoning: str`, `created_at: datetime`.
- `EscalationRecord` — `escalation_id: str`, `conversation_id: str`, `reason_code: str`, `severity: Literal["low","medium","high"]`, `created_at: datetime`.
- `PendingApproval` — `approval_id: str`, `conversation_id: str`, `candidate_decision: RefundDecision`, `required_approver_role: str`, `created_at: datetime`, `resolution: Literal["approved","denied",None] = None`, `approver: str | None = None`, `resolved_at: datetime | None = None`.

`IncidentRecord` already exists in `app/domain/models.py` (foundation commit) and is consumed unchanged.

## SQLite schema

Tables owned by this layer (migration `0001_initial.sql`):

| Table | Primary key | Unique business key | Purpose |
|---|---|---|---|
| `refunds` | `refund_id` | `(conversation_id, order_id)` | Issued refunds |
| `escalations` | `escalation_id` | — | Human-handoff records |
| `incidents` | `incident_id` (UUID) | — | Verification / HITL / tool-failure records |
| `pending_approvals` | `approval_id` | — | Awaiting-human-approval rows |
| `processed_incidents` | `incident_id` | — | Distiller idempotency marker |
| `schema_versions` | `version` | — | Applied migration versions |

LangGraph owns its own tables (`checkpoints`, `writes`) inside the same SQLite file via `SqliteSaver`. The Repository never touches those tables directly.

## Behaviors

- **Migrations** are SQL files in `app/state/migrations/`, applied in lexicographic order by `runner.apply_all()`. Each application records its `version` (filename without `.sql`) in `schema_versions`. Subsequent runs skip already-applied versions — idempotent.
- **No alembic**, no Django ORM, no Pydantic-to-DDL magic. Just plain SQL files and a 30-line runner.
- **All writes are parameterized** — `cursor.execute("INSERT INTO refunds (...) VALUES (?, ?, ...)", values)`. No f-strings into SQL. This is enforced by a unit test that grep-asserts the source.
- **`save_refund` is insert-or-noop** on the unique business key `(conversation_id, order_id)`. If a record exists, it is returned unchanged (the existing `refund_id` is preserved). This is the idempotency contract that `issue_refund` relies on for safe retry.
- **`save_incident` is insert-or-noop** on `incident_id` (UUID generated by the caller — typically the L9 verification pipeline). Safe under distiller replay.
- **`resolve_approval` is conditional-update** — only updates rows where `resolution IS NULL`. Returns the updated `PendingApproval`. Raises `ValueError("approval already resolved")` if the row is already resolved.
- **Concurrent writes serialize** under SQLite's default WAL journal mode. The test asserts this by issuing 5 concurrent `save_refund` calls via `asyncio.gather` and confirming all rows land without corruption.
- **Connection lifecycle** — a module-level singleton `sqlite3.Connection` opened with `check_same_thread=False` and WAL mode enabled. Closed by `app/api/lifespan` on shutdown.

## Events emitted

- `L5_STATE / write_performed` — payload: `{table: str, business_key: str}` — one emit per successful write.
- `L5_STATE / migration_applied` — payload: `{version: str}` — one emit per migration application.

## Tests

`tests/test_state.py` — minimum tests:

1. `test_migrations_apply_on_clean_db` — fresh tmp_path DB; `apply_all()` creates all 6 tables.
2. `test_migrations_idempotent_on_rerun` — `apply_all()` twice; no error, `schema_versions` has one row per migration.
3. `test_save_refund_inserts_new` — round-trip a `RefundRecord`, assert `find_refund` returns equal.
4. `test_save_refund_returns_existing_id_on_duplicate` — call `save_refund` twice with same `(conversation_id, order_id)`; assert the second call returns the FIRST record's `refund_id` and amount.
5. `test_find_refund_returns_none_when_missing` — query unknown key.
6. `test_save_escalation_writes_row` — round-trip an `EscalationRecord`.
7. `test_save_incident_idempotent_on_uuid` — same `incident_id` twice; one row in DB.
8. `test_list_pending_approvals_returns_unresolved_only` — insert 3 (2 unresolved, 1 resolved); list returns 2.
9. `test_resolve_approval_marks_resolved_with_approver` — assert resolution + approver + resolved_at populated; second resolve raises `ValueError`.
10. `test_checkpointer_resumes_state_across_session` — write LangGraph state under `thread_id="t1"`, close, reopen, confirm `get_tuple` returns the prior state.
11. `test_write_performed_event_emitted_per_write` — capture emitted `LayerEvent`s via a fake emitter; assert one per successful write.
12. `test_concurrent_writes_serialize` — 5 `asyncio.gather`'d `save_refund` calls with distinct keys; all 5 rows present, no `OperationalError`.

Plus the 3 new domain-model tests appended to `tests/test_domain_models.py` (round-trip Pydantic serialization for `RefundRecord`, `EscalationRecord`, `PendingApproval`).
